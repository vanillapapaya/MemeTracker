# 기술 아키텍처

## 시스템 구조도

```
[데이터 소스]          [수집/처리]              [저장]              [서빙]              [클라이언트]

X API ─────────┐
Safebooru API ─┼──→ 수집 워커 ──→ 이미지 처리 ──→ R2 (썸네일)       ┌─→ 검색 UI
브라우저 확장 ──┤      │              │              Qdrant (벡터)  ──┤   (자연어 입력)
유저 업로드 ───┘      │              │              PostgreSQL ─────┤
                      ▼              ▼              (메타데이터)     └─→ 피드 UI
               메타데이터 추출   CLIP 벡터화                            (스와이프)
               태그 정규화      (+ 선택적 캡셔닝)                         │
                                                                      광고 삽입
```

## 기술 스택

### 백엔드
| 구성요소 | 기술 | 선택 이유 |
|---------|------|----------|
| 프레임워크 | **FastAPI** (Python) | 비동기 지원, ML 라이브러리 호환 |
| 벡터 DB | **Qdrant** | 무료, 로컬 가능, 메타데이터 필터링 지원 |
| RDB | **PostgreSQL** | 유저, 메타데이터, 좋아요 등 관계형 데이터 |
| 캐시 | **Redis** | 추천 결과 캐싱, 세션 관리 |
| 태스크 큐 | **Celery** 또는 **ARQ** | 이미지 처리, 벡터화 비동기 작업 |

### AI/ML

|---------|------|----------|

## 벡터 검색 구조 (하이브리드)

### 두 가지 벡터 공간

**1. CLIP 벡터 (시각 유사도)**
- 모든 이미지에 대해 생성
- 차원: 512 (ViT-B/32 기준)
- 용도: "이 이미지와 비슷한 이미지 찾기", 텍스트 → 이미지 교차 검색
- 장점: 캡셔닝 불필요, 비용 0

**2. 텍스트 벡터 (의미 유사도)**
- 캡션, 태그, 원본 게시글 텍스트로 생성
- 모델: multilingual-e5-large, 차원: 1024
- 용도: 한국어 자연어 검색, 밈 맥락 검색
- 장점: "ㅋㅋ 레전드" 같은 한국어 맥락 이해

### 검색 흐름
```
사용자 쿼리: "뒤돌아보면서 웃는 남자"
    │
    ├──→ CLIP 텍스트 인코더 → CLIP 벡터 공간 검색 → 결과 A (시각 유사)
    │
    └──→ e5-large 인코더 → 텍스트 벡터 공간 검색 → 결과 B (의미 유사)

    결과 A + 결과 B → 점수 가중 합산 (alpha * A + (1-alpha) * B) → 최종 결과
```

### 추천 피드 벡터
```
유저 프로필 벡터 = mean(최근 좋아요/검색한 짤의 CLIP 벡터)

추천 = Qdrant.search(
    vector=유저_프로필_벡터,
    filter={"nsfw": false, "NOT_IN": 이미_본_짤},
    limit=20
)

최종 피드 = 추천 결과 70% + 랜덤 탐색 30%
```

## 데이터 모델 (PostgreSQL)

### images 테이블
```sql
CREATE TABLE images (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source          VARCHAR(20) NOT NULL,  -- 'twitter', 'safebooru', 'danbooru', 'manual', 'upload'
    source_id       VARCHAR(255),          -- 원본 플랫폼의 ID
    source_url      TEXT NOT NULL,          -- 원본 게시물 URL
    image_url       TEXT,                   -- 원본 이미지 URL (깨질 수 있음)
    thumbnail_key   VARCHAR(255),          -- R2 썸네일 키

    -- 메타데이터
    tags            TEXT[],                -- 태그 배열
    caption         TEXT,                  -- LLM 생성 캡션 (있는 경우)
    source_text     TEXT,                  -- 원본 게시글 텍스트 (트윗 등)

    -- 작가 정보
    creator_id      UUID REFERENCES creators(id),

    -- 분류
    is_nsfw         BOOLEAN DEFAULT false,
    content_type    VARCHAR(20),           -- 'meme', 'illustration', 'photo'

    -- 벡터 상태
    clip_indexed    BOOLEAN DEFAULT false,
    text_indexed    BOOLEAN DEFAULT false,

    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### creators 테이블
```sql
CREATE TABLE creators (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    display_name    VARCHAR(255),

    -- 연동 계정
    twitter_handle  VARCHAR(50),
    pixiv_id        VARCHAR(50),

    -- 프로필
    bio             TEXT,
    profile_image   VARCHAR(255),          -- R2 키
    is_verified     BOOLEAN DEFAULT false, -- OAuth 인증 여부

    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### users 테이블
```sql
CREATE TABLE users (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email           VARCHAR(255) UNIQUE,

    -- 인증
    auth_provider   VARCHAR(20),           -- 'google', 'twitter'
    auth_id         VARCHAR(255),
    is_adult_verified BOOLEAN DEFAULT false,

    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### user_interactions 테이블
```sql
CREATE TABLE user_interactions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id),
    image_id        UUID REFERENCES images(id),
    action          VARCHAR(20) NOT NULL,  -- 'view', 'like', 'search_click', 'share'
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_interactions_user ON user_interactions(user_id, created_at DESC);
```
