# 데이터 파이프라인

## 데이터 소스 및 우선순위

| 우선순위 | 소스 | 수집 방식 | 캡셔닝 필요 | NSFW 리스크 | 시점 |
|---------|------|----------|-----------|-----------|------|
| 1 | X (트위터) | 공식 API (Basic $100/월) | 낮음 (트윗 텍스트 활용) | 낮음 (필터 가능) | 초기 |
| 2 | 브라우저 확장 | 반자동 수집 (유저 클릭) | 높음 (맥락 의존) | 없음 (수동 선별) | 초기 |

## 수집 파이프라인

### 1. X (트위터) API 수집

```python
# 수집 대상: 밈 계정, 인기 짤 계정
# API: Twitter API v2 (Basic Plan, $100/월)
# 제한: 월 10,000 트윗 읽기

수집 흐름:
1. 밈 관련 계정 리스트 관리 (수동 큐레이션)
2. 타임라인/검색 API로 이미지 포함 트윗 수집
3. 트윗 텍스트 → source_text에 저장
4. 이미지 URL(pbs.twimg.com) → image_url에 저장
5. 이미지 다운로드 → 썸네일 생성 → R2 업로드
6. CLIP 벡터화 + 텍스트(트윗 내용) 벡터화
7. Qdrant에 인덱싱
```

**X API 주요 필드 매핑**:
```
트윗 텍스트 → images.source_text
트윗 URL → images.source_url
미디어 URL → images.image_url
작성자 핸들 → creators.twitter_handle
```

### 2. 브라우저 확장 프로그램 (Chrome Extension)

```
사용 흐름:
1. 사용자(기획자)가 DC/에펨코리아 등을 일반 브라우징
2. 마음에 드는 짤 위에서 확장 프로그램 버튼 클릭
3. 확장 프로그램이 자동 추출:
   - 이미지 URL
   - 게시글 제목
   - 댓글 상위 3개 (맥락 정보)
   - 게시글 URL
4. 수집 서버로 전송
5. 서버에서 이미지 다운로드 → 썸네일 → 벡터화

기술 구현:
- Chrome Extension Manifest V3
- Content Script로 DOM에서 데이터 추출
- Background Service Worker에서 서버 API 호출
- 수집 버튼: 현재 보고 있는 이미지 수집
- 일괄 수집: 갤러리 페이지에서 체크박스 선택 후 일괄 전송
```

**주의**: 자동 순회/자동 수집 기능 절대 금지. 반드시 사용자의 능동적 클릭에 의해서만 트리거.

### 4. 유저 업로드

```
업로드 흐름:
1. 사용자가 이미지 파일 또는 이미지 URL 입력
2. 이미지 유효성 검증 (크기, 형식, 안전성)
3. NSFW 필터링 (Falconsai 모델)
4. 중복 체크 (CLIP 벡터 유사도 > 0.95이면 중복)
5. 썸네일 생성 → R2 업로드
6. CLIP 벡터화
7. 사용자가 입력한 설명/태그 → 텍스트 벡터화
```

## 이미지 처리 파이프라인

### 썸네일 생성
```
원본 이미지
    │
    ├──→ 검증: 형식(jpg/png/gif/webp), 크기(10MB 이하)
    │
    ├──→ NSFW 체크: Falconsai/nsfw_image_detection
    │    ├── safe → 계속
    │    └── unsafe → 거부 또는 NSFW 플래그
    │
    ├──→ 리사이즈: 200x200 (검색 결과용)
    │              400x400 (피드용)
    │              원본 비율 유지, 크롭
    │
    └──→ R2 업로드
         키: thumbnails/{source}/{YYYY-MM}/{uuid}.webp
         형식: WebP (용량 최적화)
```

### 벡터화
```
이미지
    │
    ├──→ CLIP 인코더 (ViT-B/32)
    │    └── 512차원 벡터 → Qdrant clip_vectors 컬렉션
    │
    └──→ 텍스트 소스 수집
         ├── booru 태그 → tags_to_caption() → 자연어 변환
         ├── 트윗 텍스트 → 직접 사용
         ├── 게시글 제목+댓글 → 직접 사용
         └── 유저 입력 설명 → 직접 사용
              │
              └──→ e5-large 인코더
                   └── 1024차원 벡터 → Qdrant text_vectors 컬렉션
```

### 선택적 LLM 캡셔닝
```
대상: 태그도 텍스트도 없는 이미지 (브라우저 확장 수집 중 메타데이터 부족한 경우)

Claude Vision API:
- 프롬프트: "이 이미지를 밈으로 사용할 때의 맥락을 포함해 한국어로 설명해주세요."
- 비용: 이미지당 약 $0.01
- 배치 API 사용 시 50% 절감

스케일 시 대안:
- InternVL2.5 또는 Qwen2.5-VL (오픈소스, 로컬 실행)
- GPU 서버 필요 (월 $50-100)
```

## 콜드 스타트 전략

### Phase 1: 수동 수집 (1주)
- 목표: 특정 카테고리 500-1,000개
- 방법: 기획자가 직접 수동 수집
- 카테고리: 글로벌 밈 (distracted boyfriend 등) + 한국 커뮤니티 인기 짤
- 비용: $0

### Phase 2: 브라우저 확장 반자동 (2주)
- 목표: 5,000-10,000개
- 방법: 브라우저 확장 프로그램 개발 후 기획자가 수집
- 속도: 짤당 3초, 하루 2시간 = 2,400개/일
- 비용: $5 (캡셔닝)

### Phase 3: API 자동 수집 (상시)
- 목표: 10,000개 이상
- 방법: Safebooru API + X API 자동 수집
- Safebooru: 일 1,000개 수집 가능
- X: 월 10,000 트윗 (Basic Plan)
- 비용: $100/월 (X API)

### Phase 4: 유저 기여 (서비스 성장 후)
- 브라우저 확장을 일반 사용자에게 배포
- 유저 업로드 기능
- DB 자연 확장 플라이휠
